The CLI should install its self so that it can be invoked simply with calli, and doing so will show the help manual

Commands listed below can either be run as calli command [argument] <--flag "option">/<-f "opt">

where [argument] is required and <--flag "option"> is optional

If a flag and its option are not specified, you will then be asked to input that option

Options MUST be surrounded by quotation marks, and the argument must not

Example

To create an event next Monday at 2pm to visit Jane Doe at 221b Baker Street for 45 minutes, the following one-liner can be used

calli event visit Jane Doe -d "monday" -t "2pm" -l "45" -i "221b baker street

event @4eb2fa created

Input allowances

--day inputs will accept any non-relative day, date-first (i.e. dd/mm/yyyy not mm/dd/yyyy)

  • sunday
  • 5jul
  • 5/7/2020
  • etc

--time inputs will accept many time formats

These times currently only obey local timezone, and get stored as UTC. changing your local timezone will offset your calendar events

  • 23:00
  • 11:00pm
  • 11pm
  • etc

Commands

event

calli event [event name]

creates a new calendar event

calli event First of June -d 1/6/2021 -t 1:30pm -l 15 -i "my favourite month"
  • --day, -d: date of the event [DD/MM/YYY]
  • --start, -t: start time of the event [HH:MMam/pm]
  • --duration, -l: duration of the event [minutes]
  • --info, -i: information for the event "(optional)"

remind

calli remind [reminder name]

creates a new reminder

calli remind Send latest report to Nick -d 24/6/2021 -t 11:00am -l 15 -i "nick@example.com"
  • --day, -d: date of the reminder [DD/MM/YYY]
  • --time, -t: time of the reminder [HH:MMam/pm]
  • --info, -i: information for the reminder "(optional)"

delete

calli remind [@hex] [@hex]

deletes an event specified with a hexcode

calli delete @4eb2fa 5f3a65

Event hex codes can be passed with or without the @
